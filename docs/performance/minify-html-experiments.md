# minify-html Experiments

This report is generated by `bun run payload:minify-html:experiments` so
the same scenarios can be rerun after site, dependency, or requirements
changes. It measures raw bytes, gzip bytes, and Brotli bytes because the
production target is static HTML served by compression-capable hosting.

## Conclusion

No tested scenario is production-eligible. Do not add `minify-html` to
the production build until a scenario passes strict validation and the
rest of the release gates.

## One-Pass Feasibility

The one-pass variant remains documented in `docs/performance/minify-html-research.md`.
This reproducible suite exercises the standard `@minify-html/node` binding
because it is the Bun-friendly cross-platform path currently available to
the project. One-pass should be added to this suite only if a maintained
Node/Bun-friendly package or CLI path is proven.
