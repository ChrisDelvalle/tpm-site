# Standalone Post-Build Optimization Experiments

This report is generated by `bun run payload:postbuild:experiments` so
standalone optimizers can be evaluated independently of Astro/Vite config
support. Each scenario operates on a copied `dist/` directory, then runs
generated-output gates and raw/gzip/Brotli payload measurement.

## Tooling Context

- `oxc-minify` can be used as a standalone Node API over generated JS;
  upstream labels the minifier as alpha and recommends thorough production
  testing.
- `lightningcss` is available as a standalone parser/minifier and is already
  part of the Vite/Astro ecosystem.
- `svgo` is a standalone SVG optimizer and Node API; this suite keeps
  `viewBox` preservation explicit.

## Conclusion

Best passing candidate by Brotli size: `safe-stack`.
This is a candidate for a production post-build step, not an automatic
adoption. It still needs the full release gate against optimized output.

## Adoption Notes

- A passing payload experiment is not enough for production adoption when
  JavaScript output changes. The full browser, accessibility, HTML, build
  verification, and release gates must pass with the optimized output.
- Static generated-output verification recognizes Astro's prefetch runtime
  by checking the stable runtime behaviors instead of exact generated-code
  quote style or whitespace.
- Measurement-only Oxc aggressive output is intentionally excluded from
  production recommendations until its assumptions are proven against this
  site's actual browser behavior.
